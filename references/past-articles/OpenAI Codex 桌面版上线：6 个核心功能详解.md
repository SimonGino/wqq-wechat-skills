## 功能 1：多项目管理

![image-20260204165517985](https://oss.mytest.cc/2026/02/e6c999a3fa3da73f4256f9226b07cdab.png)

Codex 桌面版支持同时添加多个项目目录。

左侧栏显示所有项目，你可以：
- 查看每个项目当前运行的任务
- 监控任务进度
- 快速切换项目

**使用场景**：你手上有多个项目，想让 AI 同时帮你处理不同任务。


还有一个贴心的设计：**集成了 Git 图形操作界面**，也可以直接打开终端。对于不熟悉命令行的用户来说，这个很友好。

## 功能 2：Worktree（工作树隔离）

![image-20260204165652686](https://oss.mytest.cc/2026/02/2fc174e137ed5a86fa680d9827e4c452.png)

这是个很实用的安全机制。

**它解决什么问题**：AI 写代码时，如果直接改你的文件，万一改错了很麻烦。

**Worktree 怎么工作**：
1. Codex 创建一个独立的代码副本（工作树）
2. AI 在这个副本里改代码
3. 完成后，你查看改动（diff）
4. 确认没问题，再合并回主代码

**好处**：
- AI 怎么折腾都不会弄乱你的本地代码
- 多个 AI 任务可以并行，互不干扰
- 你有完全的审核权

---

## 功能 3：Plan Mode（计划模式）

![image-20260204165909761](https://oss.mytest.cc/2026/02/5ec46bff86dab1f33cf1e4118b094f40.png)

这个功能让你对 AI 的行为有更好的控制。

**传统模式的问题**：你告诉 AI 要做什么，它直接开始改代码。有时候它理解错了，改完你才发现不对。

**Plan Mode 怎么工作**：
1. 你下达任务
2. AI 先告诉你它打算怎么做（计划）
3. 你审查计划，可以提修改意见
4. 确认后，AI 才开始执行

**什么时候用**：
- 任务比较复杂
- 你想确保 AI 理解对了
- 涉及重要代码，不想让 AI 乱改

---

## 功能 4：沙盒模式（Sandbox）

![image-20260204165942549](https://oss.mytest.cc/2026/02/94d66cdd4a8cedbcd7e38c422fe12910.png)Codex 提供两种权限模式来控制 AI 能做什么。

**只读模式（Sandbox）**：

- AI 不能修改文件
- AI 不能执行命令
- AI 不能联网
- 适合：让 AI 分析代码、回答问题

**完全访问模式（Full Access）**：
- AI 可以改文件、跑命令、联网
- 适合：让 AI 实际写代码、跑测试

**建议**：先用只读模式让 AI 分析，确认方案后再开 Full Access。

## 功能 5：Automations（定时任务）

![image-20260204170023464](https://oss.mytest.cc/2026/02/7dcad1e043e47e67e6547c0c5121aa52.png)

这个功能让 AI 变成你的"值班员"。

**它能干什么**：
- 定期扫描代码找潜在 bug
- 从合并的 PR 自动写 release notes
- 总结昨天的 git 活动
- 汇总 CI 失败和测试问题
- **自动写日报**——有用户说这个功能"很舒服"

**工作机制**：
- 有发现才通知你，没事就静默归档
- 默认用 Worktree 隔离，不影响你的主工作区
- 结果进收件箱，你只做最终决策

**注意事项**：
- 目前需要 APP 开着才能运行
- 项目目录必须在本地
- 云端定时器还在开发中

## 功能 6：Skills（技能卡片）

![image-20260204170014321](https://oss.mytest.cc/2026/02/815efd45a2fb038859f05589eb729346.png)

Skills 把常用操作封装成可复用的模块。

**怎么理解**：就像把"怎么做某件事"写成一张操作卡片，下次直接用。

**安装方式超简单**：
- 图形化界面点击安装
- 或者用自然语言让它帮你安装（比如说"帮我装一下 Figma skill"）

**官方提供的 Skills**：
- **Figma**：把设计稿转成代码
- **Linear**：管理项目和任务
- **Vercel/Netlify/Cloudflare**：部署相关
- **办公类**：读写 PDF、表格、Word 文档

**自定义 Skills**：
- 你可以创建团队专属的技能
- 用 Skill Creator 教 AI 怎么用你们的内部 API
- Skills 可以和定时任务配合使用

**触发方式**：
- 显式调用：在指令里点名 `$skill-name`
- 隐式调用：AI 自动判断该用哪个技能

**为什么 Skills 很重要**：正是 Skills 的成熟，让 Codex 从单纯的"编程工具"变成了通用的"AI 工作助手"。写文章、做编辑、搞投研，都可以通过 Skills 来实现。

## 如何开始使用

**第一步：下载安装**
- 访问 openai.com/codex
- 下载 macOS 桌面应用
- 用 OpenAI 账号登录

**第二步：添加项目**
- 打开 APP
- 添加你的项目目录
- 项目会显示在左侧栏

**第三步：开始使用**
- 选择一个项目
- 用自然语言描述你想做什么
- 选择是否使用 Worktree 模式
- 查看结果，确认后合并

## 常见问题

**Q：免费用户能用吗？**
A：可以。限时免费试用 2 个月。

**Q：需要会写代码吗？**
A：不需要。用自然语言描述就行。

**Q：会弄乱我的代码吗？**
A：用 Worktree 模式就不会。AI 在独立副本里工作，你审核后才合并。

**Q：定时任务需要电脑一直开着吗？**
A：目前是的。云端定时器还在开发中。

**Q：和 Claude Code 有什么区别？**
A：Codex 更强调"撒手"模式，把任务扔给 AI 自己跑完再汇报。Claude Code 更强调实时互动。
